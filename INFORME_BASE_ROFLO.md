# Informe base - Proyecto Integrador ROFLO Cheesy Nach

## 1. Introducción

El negocio seleccionado es ROFLO Cheesy Nach, un emprendimiento de comida rápida especializado en nachos con queso, toppings, combos y promociones. El propósito de la aplicación móvil es permitir que los clientes consulten el menú, personalicen productos, realicen pedidos y puedan dar seguimiento a su orden. Además, la aplicación incluye un módulo administrativo para gestionar clientes y productos.

## 2. Análisis del proyecto

Se analizaron las necesidades principales del negocio: mostrar productos de forma atractiva, facilitar la selección de tamaños y extras, permitir el registro de clientes, llevar control de pedidos y ofrecer un panel administrativo. También se tomó como referencia el estilo visual de aplicaciones de comida rápida, priorizando botones claros, imágenes grandes, navegación inferior y colores de marca.

## 3. Diseño de la aplicación

El diseño mantiene la identidad visual de ROFLO: fondo oscuro, naranja, amarillo queso, tarjetas redondeadas, imágenes grandes de productos y encabezados con efecto de queso derretido. La app cuenta con pantallas de splash, login, registro, inicio, menú, detalle de producto, carrito, pedidos, perfil, panel administrador, clientes y productos.

## 4. Desarrollo de la aplicación

La aplicación fue desarrollada en Flutter. Se implementaron modelos de datos para productos, carrito, clientes y pedidos. Las funcionalidades principales son: navegación por menús, visualización de productos, personalización de producto, carrito, confirmación de pedido, seguimiento de pedidos y CRUD de clientes. También se agregó un CRUD básico de productos para el panel de administración.

## 5. Pruebas y depuración

Se recomienda probar: inicio de sesión, registro, navegación entre pantallas, agregar producto al carrito, modificar cantidades, confirmar pedido, crear cliente, editar cliente, eliminar cliente, crear producto y editar producto. Los errores encontrados durante la prueba deben documentarse con captura de pantalla y explicación de la solución aplicada.

## 6. Generación del APK

Para generar el APK se debe ejecutar el comando `flutter build apk --release`. Luego se debe capturar el proceso en consola y la ubicación final del archivo APK.

## 7. Conclusiones

El proyecto permitió aplicar conceptos de diseño UI/UX, navegación móvil, manejo de estado, formularios, listas dinámicas y operaciones CRUD. Como mejoras futuras se puede implementar base de datos local o en la nube, autenticación real, pasarela de pagos y notificaciones para el seguimiento de pedidos.
